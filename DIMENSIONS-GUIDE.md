## 图三：使用时机决策流程图

> 遇到问题时，照着这棵树往下走，找到该用哪个命令。

```
你遇到了什么问题？
│
├─► 【我不确定现在哪里最需要提升】
│       └─► 运行 /vibe-startup  → 自动评分全部10维度，告诉你最低分在哪
│
├─► 【法律 / 平台政策 / 合规方面的问题】
│       ├─ 微信封号警告 / 广告被拒 / 营业执照问题
│       ├─ 合同纠纷 / 隐私政策 / 数据合规
│       └─► 运行 /vibe-legal
│
├─► 【钱的问题】
│       ├─ 烧钱太快 / 不知道还能撑多久
│       ├─ 单个用户成本(CAC)搞不清楚
│       ├─ 毛利率 / 财务模型 / 融资准备
│       └─► 运行 /vibe-finance
│
├─► 【产品 / 技术的问题】
│       ├─ 功能不完整 / Bug太多 / 性能差
│       ├─ 用户体验差 / 流程不顺畅
│       ├─ 小程序发版 / 代码架构
│       └─► 运行 /vibe-production
│
├─► 【看不清楚业务状况 / 数据问题】
│       ├─ 不知道用户在哪里流失
│       ├─ 没有转化漏斗 / 没有留存数据
│       ├─ KPI靠人工Excel整理，费时费力
│       └─► 运行 /vibe-data
│
├─► 【客服 / 用户投诉问题】
│       ├─ 回复太慢 / 用户抱怨没人理
│       ├─ 重复问题太多，客服累死
│       ├─ 满意度低 / 差评多
│       └─► 运行 /vibe-service
│
├─► 【用户来了留不住 / 活跃度低】
│       ├─ 新用户注册了但不用
│       ├─ 老用户慢慢消失 / 复购率低
│       ├─ 没有社群 / 没有VIP体系
│       └─► 运行 /vibe-operations
│
├─► 【推广 / 获客 / 品牌问题】
│       ├─ 不知道在哪里推广
│       ├─ 投了广告没效果
│       ├─ 内容没人看 / 公众号没粉丝
│       ├─ 视频号 / 小红书 / KOL合作
│       └─► 运行 /vibe-marketing
│
├─► 【增长遇到瓶颈 / 想规模化】
│       ├─ 增长停滞 / 不知道下一步往哪发力
│       ├─ 想搞清楚北极星指标
│       ├─ CAC太高 / LTV:CAC比例不健康
│       ├─ 想建立口碑传播 / 老带新
│       └─► 运行 /vibe-growth
│
├─► 【团队 / 人员 / 协作问题】
│       ├─ 职责不清 / 重复劳动 / 互相推诿
│       ├─ 没有SOP / 新人上手慢
│       ├─ 工具太乱 / 效率低
│       └─► 运行 /vibe-team
│
└─► 【重复性工作太多 / 想用AI提效】
        ├─ 手动整理报告 / 手动回复用户 / 手动发内容
        ├─ 想用AI做客服 / 做内容 / 做数据分析
        ├─ 流程自动化 / 系统之间打通
        └─► 运行 /vibe-automation
```

## 速查卡：一句话记住每个维度

| 命令 | 一句话定义 | 最常用的时候 |
|------|-----------|-------------|
| `/vibe-legal` | **"不出问题"** — 合规是生死线 | 上线前、被警告时、新功能涉及敏感数据时 |
| `/vibe-finance` | **"活下去"** — 现金流是命脉 | 月底对账、融资前、投放前估算ROI |
| `/vibe-production` | **"做好产品"** — 功能稳定是基础 | 版本迭代前、性能问题、用户体验差时 |
| `/vibe-data` | **"看清楚"** — 没数据就是盲飞 | 不知道哪里流失、要做决策时 |
| `/vibe-service` | **"留住用户"** — 服务是留存护城河 | 差评增多、客服压力大、响应慢时 |
| `/vibe-operations` | **"用户生命周期"** — 激活→留存→VIP | 活跃度下降、复购率低时 |
| `/vibe-marketing` | **"让人知道你"** — 流量是增长的燃料 | 用户增长停滞、想做推广时 |
| `/vibe-growth` | **"规模化复制"** — 把增长变成系统 | 增长瓶颈、想扩张时 |
| `/vibe-team` | **"让团队能跑"** — 人和流程决定上限 | 新人入职、协作混乱、SOP缺失时 |
| `/vibe-automation` | **"10倍效率"** — 用AI做人做的事 | 重复劳动多、想提效时 |
| `/vibe-startup` | **"全局视角"** — 哪里最弱就补哪里 | 定期复盘、不知从何下手时 |
